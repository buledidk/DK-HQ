# Budget & Forecast — Evaluation, Reasoning, Analysis

Cross-sector framework to challenge, rebuild, and communicate budgets and forecasts. Depth in financial services; breadth across all 10 global sectors and UK industry verticals.

## When to use

- Annual budget review / board sign-off.
- Mid-year forecast reset.
- Zero-based budgeting programme.
- Driver-based FP&A redesign.
- Long-range plan (LRP) for investors / regulators.
- Variance analysis (actual vs budget vs prior forecast).
- Scenario / stress / reverse stress testing.
- Going-concern model + covenant headroom.

## Evaluation framework — the 7-lens challenge

Every budget / forecast passes all 7:

| Lens | Test | Pass = |
|---|---|---|
| **1. Structural** | 3-statement integrity (P&L → BS → CF ties) | No plug, no circular, opening = prior closing |
| **2. Driver** | Each line driven by volume × price / rate / ratio — not growth % | ≥ 80% of P&L lines have explicit drivers |
| **3. External** | Market, macro, regulator assumptions consistent with external data | Sources cited; GDP, inflation, rate-path from credible house |
| **4. Internal** | Consistent with strategy, capex plan, headcount plan | No orphan assumptions; strat doc + FP&A in sync |
| **5. Sensitivity** | Key drivers flexed ± 10% / ± 20% | Material outputs disclosed; top 3 drivers identified |
| **6. Historical** | Forecast accuracy reviewed vs last 3 cycles | Bias quantified; any systematic optimism corrected |
| **7. Credibility** | Does management's track record support the numbers? | Backed by cohort / pipeline / run-rate evidence |

## Reasoning — the "because" chain

Forecasts must be defensible under ISA 540 / ISA 570 / regulator challenge. Apply this chain to every material assumption:

1. **Claim** — e.g. "Revenue grows 14% YoY."
2. **Driver** — volume 8% + price 6%.
3. **Evidence** — signed backlog (40% cover), pipeline conversion history (6-year median), pricing study vs peers.
4. **Counter** — what would have to be true for this to be wrong? (competitor entry, churn spike, regulatory cap).
5. **Sensitivity** — range with probabilities.
6. **Conclusion** — reasonable / optimistic / aggressive / speculative.

Anything labelled "management view" without the chain = red flag.

## Financial services — deep lens

Specific FS forecast challenges (you will see most of these every cycle):

### Banking forecast pitfalls
- NII projected off a single rate path → always require rate-shock grid (-200/-100/base/+100/+200 bps; parallel + non-parallel).
- Deposit beta assumptions static through a cycle → floor rates once reached.
- Loan growth > system loan growth → require explicit market-share theory.
- ECL coverage stable despite stage-2 migration → model broken.
- NIM forecast without forward curve → back to drawing board.
- Cost/income ratio "to reduce by 200bps" with no specific cost action → no.

### Insurance forecast pitfalls
- Claims ratio projected at 5-yr average with no cat loading → weather/large-loss events omitted.
- LDI / matching adjustment base without spread-widening stress.
- IFRS 17 CSM unwind pattern assumed but not walked through.

### Asset management forecast pitfalls
- Net flow = gross inflow + redemption assumed separately inconsistent with AUM base.
- Fee margin decline of 1bp p.a. stopped arbitrarily → passive shift ongoing.
- Performance-fee crystallisation modelled at 100% across multi-year HWM → improbable.

## Zero-based budgeting (ZBB)

When to apply:
- Cost-out programme > 10% of base.
- New management team, new strategy, post-M&A.
- Loss-making business unit.

ZBB playbook:
1. **Activity inventory** — every activity by cost bucket.
2. **Customer / outcome map** — each activity's purpose and value.
3. **Frequency + capacity** — how often, how many FTE / £.
4. **Challenge tiers** — eliminate / reduce / retain / invest.
5. **Target operating model** — re-stack at lower cost with controls.
6. **Governance** — tracker, savings P&L flagging, claw-back discipline.

## Driver-based FP&A — target state

- Every P&L line driven by volume × rate (not % growth).
- Drivers sourced from system-of-record (CRM for pipeline, HRIS for FTE, billing for ARR).
- One-source model (no desktop overrides that vanish next cycle).
- Rolling 18-month horizon (13 months gets you short-term; 18 gives you next-year crossover).
- Monthly re-forecast, quarterly board view, full reset mid-year.

## Variance analysis — audit-grade

Standard format:

| Line | Actual | Budget | Var £ | Var % | Driver-led explanation |
|---|---|---|---|---|---|
| Revenue | 1,240 | 1,180 | +60 | +5.1% | Volume +3%, price +2%, mix flat |
| COGS | (820) | (780) | -40 | -5.1% | Volume -3%, input cost +2% |
| Gross margin | 420 | 400 | +20 | +5.0% | Mostly volume flow-through; price held |

Required: explanations are driver-level, not label-level ("market conditions" is not an explanation).

## Scenario / stress / reverse stress

| Test | Purpose |
|---|---|
| **Base** | Management expectation |
| **Upside** | Top of credible range |
| **Downside** | Bottom of credible range |
| **Stress** | Plausibly severe (e.g., 1-in-25) |
| **Reverse stress** | What would break the firm? What conditions produce it? |

Banks + systemic insurers must run reverse stress under SS20/15 / Solvency. Others should run it because it's the most diagnostic of all.

## Deliverables

- Budget challenge memo (7-lens assessment + top 10 issues).
- Re-cut forecast (driver-based, model evidenced).
- Variance report (line × driver × owner).
- Scenario pack (base / upside / downside / stress / reverse stress).
- KPI dashboard (5-7 metrics per business, peer-benchmarked).
- Board briefing (one page + appendix).

## Anti-patterns

- "Budget is top-down from the CEO" with no bottom-up reconciliation.
- Forecasts always beaten by actuals → sandbagging; always missed → optimism.
- Every year shows hockey-stick uplift in Q4 / H2 with nothing in Q1.
- Cost savings show in P&L with no people / systems / contract lever identified.
- Strategic narrative talks transformation; forecast looks like last year +3%.
- Sensitivity tables without probability or narrative.
