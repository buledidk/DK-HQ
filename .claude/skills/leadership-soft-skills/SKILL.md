# leadership-soft-skills

The human side of delivering everything the other skills describe.

## Communication — by audience

### Universal rules
- Lead with the ask, then the context
- Plain English; define jargon once or not at all
- Signal confidence: "80% sure because X, 20% worried about Y"
- State assumptions before conclusions
- Active voice; short sentences
- Say the quiet part out loud

### Board / AC / Regulator
- Third-person neutral, evidence-first
- BIG-4 pattern (Bottom line → Implications → Ground truth → 4th ask)
- Know the 3 tests: fact, judgement, escalation
- When you don't know, say so + commit to when you'll know
- Never surprise the chair — pre-brief

### Executive / C-suite
- 1 page or 5 slides max
- Headlines before details
- Commercial impact front and centre
- Give an opinion + option to reject

### Team (direct + matrixed)
- Why before what
- Context + constraints + criteria for success = autonomy
- Repeat priority order weekly
- Celebrate specific behaviours
- Async, permanent, searchable

### Client / Investor / External
- Commercial voice, first-person plural
- Promise-keeping > promise-making
- Forward-looking always qualified
- MAR / inside-info discipline

### Difficult conversations (performance, conflict, bad news)
- Private, direct, specific, timely
- **SBI**: Situation + Behaviour + Impact
- State facts, impact, ask, support
- Silence is fine — don't fill it
- Agree next step + date in writing

### 7-minute test
- Reader gets the point in 30 seconds
- Decides in 7 minutes
- Appendices for depth

## Decision-making under uncertainty

1. Name the decision — what exactly?
2. Reversibility — one-way or two-way door?
3. Time-to-decide — cost of "too slow"?
4. Info value — what would change the answer?
5. Bar — satisficing vs optimising?
6. Owner + date
7. Communicate — decision + rationale + review triggers

Under genuine uncertainty:
- Reduce blast radius (pilot, staged, reversible)
- Two-way door → decide quickly; one-way door → decide carefully
- Pre-mortem
- Disagree + commit
- Known unknowns explicit

Traps: analysis paralysis, sunk-cost, optionality theatre, groupthink,
recency bias.

Escalate if: beyond delegated authority, irreversible / high blast radius,
conflicts with prior exec decision, cross-function trade-off, regulatory
exposure, time-sensitive beyond your horizon.

## Distributed teams (countries + cultures + time zones)

### Operating model
- Default async; sync must justify itself
- Written > verbal; decisions in writing
- Core overlap 2–4 hrs protected for live collab
- Follow-the-sun with structured briefs
- Regional anchors with local decision rights
- Universal daily stand-up doc instead of standups

### Rituals
- Weekly team pulse (rotating owner)
- Monthly skip-level 1:1s across geographies
- Quarterly in-person where possible
- Fortnightly client / stakeholder sync
- Monthly retrospective with written decisions

### Cross-cultural (calibrate, don't stereotype)
- Directness of feedback
- Hierarchy
- Task vs relationship
- Time orientation (monochronic vs polychronic)
- Context (low vs high)

Rules:
- Ask, don't assume — "what's the best way to disagree here?"
- Adapt style; insist on outcomes
- Written feedback travels well; spoken poorly
- Avoid idioms in writing
- Recognise holiday + religious calendars

### Time-zone discipline
- Meeting time rotates — no single region always takes the late call
- Every meeting: agenda + pre-read + recording + notes
- Decisions in writing within 24 hrs
- Never page outside hours for non-critical

### Trust at distance
- Trust = reliability × time
- Credit in public; critique in private
- Visibility of work, not presence
- Travel budget = leadership tool

### Tooling
- Notion / Coda / Confluence as SOT
- Slack / Teams / Loom for async
- Jira / Linear for work tracking
- Zoom / Meet with recording + transcripts (consent)

## Team management lifecycle

### Hiring
- Job scorecard (outcomes + competencies + culture add)
- Structured interviews, scored
- Diverse panel; one "no" → "no" unless justified
- Reference checks (3 for senior)
- 30/60/90-day plan before day 1

### Onboarding
- Buddy + manager + skip-level first week
- Context before tasks
- Shadow → supported → independent
- Feedback week 2, 4, 8
- Docs-as-you-go

### Performance management
- Clear goals (OKRs) + weekly check-ins
- Continuous specific feedback (SBI)
- Formal review 2×/year
- Calibration across peers
- Problems surfaced early

### Performance conversations
- Underperformance: expectations + measurable targets + time-bound +
  documented support + consequences
- High performers: growth + stretch + visibility + retention
- Middle 70%: engagement + development drives team performance

### Letting go (last resort)
- Documented support + time
- HR + legal
- Dignity + clarity + practical support
- Brief factual team comms; no gossip
- Self-reflection on systemic failure

## Productivity + yield

### Team-level
- Clarify priorities weekly
- Kill work that doesn't matter
- Remove blockers before they compound
- Measure throughput (cycle time, first-pass yield, touchless %) not hours
- Design for exception, not every case

### Individual-level
- Deep work blocks (2–3 hrs)
- One big thing per day
- Energy > time management
- Write more, meet less
- End-of-day plan for tomorrow's first hour

### Organisational
- Budget buffer for quality (FRC AQR: time pressure = quality risk)
- Metrics for the metrics
- Post-implementation review = learning loop
- Pre-mortem new initiatives

## Ambiguity navigation

### Early days
- Facts + assumptions + unknowns in writing
- 5–10 conversations before forming opinion
- Steel-man opposite view
- Smallest disambiguating experiment
- Decision date

### At a fork
- 1 paragraph per path (destination + cost + risk)
- Reversible vs irreversible per path
- Pick, communicate, commit, review at next decision point

### Everyone disagrees
- Facts, values, or method?
- Facts → data; values → explicit trade-off; method → pilot
- Undecidable? Leader decides + owns + commits to revisit

### Don't know what to do
- Say so
- Reduce to next reversible step
- Time-box ("by Friday I'll know whether X")

## Executive presence

- Calm tone — especially when others aren't
- Short, clear answers first; depth on demand
- Acknowledge the question before answering
- Land planes: topic → conclusion → next step
- Admit error + commit to fix
- Humour sparingly, never at others' expense
- Remember names + context — signals respect

## Stakeholder navigation

### Map
- Power × Interest grid
- Per stakeholder: decision rights, success criteria, channel, concerns
- Engagement cadence
- Pre-brief before big meetings (no surprises)
- Post-brief after key decisions

### Negotiation
- Separate people from problem
- BATNA — yours + theirs
- Interests not positions
- Options for mutual gain
- Objective criteria
- Walk away if long-term relationship damaged

### Conflict resolution
- Listen 2:1
- Name the conflict
- Shared goal first
- Trade-offs + constraints next
- Next step + review date

## Milestones + delivery

- Outcome milestones — KPI movement, not task ticks
- Sponsors see progress weekly
- Pre-mortem every significant deliverable
- RAID log weekly; risks have owners + mitigations
- Go-live gate — rollback + hypercare + KPI monitoring
- PIR — actual vs business case + benefit realisation

## Red flags

- More meetings than output
- Decisions constantly revisited without new info
- "Everyone is fine" without evidence (psychological safety issue)
- Key-person dependency
- Cross-border friction ignored
- Burnout signals: silence where there was voice, late emails, quality drops
- You are bottleneck on everything

## Output pattern

Weave in:
- Audience-appropriate communication
- Explicit decision framework
- Cultural + time-zone awareness
- Acknowledged uncertainty + reduction path
- Concrete next step with owner + date
