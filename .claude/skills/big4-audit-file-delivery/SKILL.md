# big4-audit-file-delivery

Industry-agnostic, Big-4-system-grade skill for preparing, drafting,
reviewing, quality-controlling, and approving external audit files end to
end — across every sector.

## Why this skill exists

External audit is the same ISAs everywhere — but the **file** is what
gets inspected by FRC AQR, PCAOB, IAASB, and firm monitoring. File
discipline + Big 4 system fluency decides whether an engagement gets a
clean regulator review.

## Audit systems

| System | Firm | Strengths |
|--------|------|-----------|
| **PwC Aura** / **Connect** | PwC | Risk-driven structure, automated procedures, analytics |
| **Deloitte Omnia** / **EMS** | Deloitte | Digital audit, Cortex analytics, AI-assisted review |
| **EY Canvas** / **Helix** | EY | Canvas as WP platform; Helix for data analytics |
| **KPMG Clara** | KPMG | Integrated global, smart analytics, automated controls |
| **BDO Advantage** | BDO | Mid-tier flagship, ISQM 1-aligned |
| **Grant Thornton Voyager** | GT | Sector-template rich, risk-driven |
| **Mazars Signature** | Forvis Mazars | Unified EU platform |
| **Caseware IDEA / Working Papers** | Mid-tier + independent | Industry standard non-Big-4 |
| **Wolters Kluwer CCH ProSystem fx** | US + UK mid-tier | US-centric |

Each differs in UI but maps to the same ISA-driven file structure. Apply
at the conceptual file level.

## Canonical file structure

### Section 00 — Engagement admin
- Engagement letter (ISA 210) + independence (FRC Ethical Std + ISQM 1)
- Acceptance / continuance memo
- Fee structure + NAS whitelist
- Team + specialists + component auditors
- EP + EQR designation (ISQM 2)
- TCWG communication plan

### Section A — Planning + strategy
- Audit strategy memo (ISA 300)
- Materiality memo (ISA 320): OM, PM, CTT, specific, component
- Risk assessment memo (ISA 315 revised)
- Fraud-risk memo (ISA 240 + brainstorming)
- Significant-risk register
- IT environment + ITGC scoping
- Planning analytics
- Component scoping (ISA 600)
- Audit plan per significant area (ISA 330)
- Sign-offs: senior, manager, EP, EQR planning stage

### Section B — Controls (TOC)
- Process narratives + walkthroughs per significant CoTABD
- Control inventory (design evaluation)
- TOC plan + evidence + evaluation
- ITGC testing (access, change, ops, acquisition)
- Application controls
- SOC 1 / ISAE 3402 (ISA 402)
- Deficiency schedule + severity + communication

### Section C — Substantive (by FS line)

| Section | FS area | Key ISAs + assertions |
|---------|---------|----------------------|
| C1 | Cash + bank | ISA 505, E + R + V |
| C2 | Investments | ISA 500 + ISA 540, E + V + O + P |
| C3 | Trade receivables | ISA 505, E + V + A + R |
| C4 | Inventory | ISA 501 observation, E + V + C |
| C5 | Other current assets | E + V |
| C6 | PP&E | ISA 540, E + V + R + C |
| C7 | Intangibles + goodwill | IAS 36 + 38, V + R + C |
| C8 | Other non-current assets | E + V |
| C9 | Deferred tax | IAS 12, V + C + P |

### Section D — Liabilities
D1 Trade payables; D2 Accruals + deferred income; D3 Tax; D4 Provisions
(IAS 37); D5 Borrowings + covenants; D6 Leases (IFRS 16); D7 Pensions
(IAS 19); D8 Financial instruments (IFRS 9); D9 Derivatives + hedges.

### Section E — Equity
Share capital + reserves, dividends, SBP (IFRS 2), treasury shares, EPS
(IAS 33).

### Section F — P&L
Revenue (IFRS 15 per stream), COGS + inventory flow, opex, other
inc/exp, finance inc/cost, tax charge reconciliation, discontinued ops
(IFRS 5).

### Section G — Other audit areas
Related parties (ISA 550); subsequent events (ISA 560); going concern
(ISA 570); laws + regs (ISA 250); estimates (ISA 540) cross-ref; JE
testing (ISA 240 ¶32); litigation (ISA 501); legal confirmations.

### Section H — Group audit (ISA 600)
Component scoping; materiality cascade; instructions; component
clearance + review notes; consolidation testing (elimination, FX, NCI);
component SUM aggregation.

### Section I — Completion
SUM (ISA 450); OI (ISA 720); written representations (ISA 580); final
analytical; auditor's report drafts + clearance; KAM drafting (ISA 701);
GC final conclusion; closing-meeting minutes; management letter.

### Section J — Quality
EQR WPs + conclusion memo (ISQM 2); consultation logs; differences of
opinion resolved; pre-issuance sign-offs.

### Section K — Archive
Assembly tracker (60-day, ISA 230 + ISQM 1 A54); retention calendar;
post-archive amendment log.

## Lead schedule (every section)

Opening + movement + closing; tie to TB/FS + to PY; cross-ref to WPs;
comments + conclusions; preparer + reviewer sign-off.

## WP standards (ISA 230)

Every WP:
1. Title
2. Purpose (linked to RoMM + assertion)
3. Source + completeness check
4. Procedure (specific, present-tense)
5. Results (findings, sample outcomes, misstatements)
6. Conclusion (addresses objective?)
7. Preparer + date
8. Reviewer + date (coaching notes closed)
9. Cross-references (forward + back)
10. Source docs attached / hyperlinked

Every WP must:
- Stand alone (readable with no prior context)
- Demonstrate sufficient appropriate evidence (ISA 500)
- Show judgement explicitly
- Link to lead-schedule conclusion
- Be free of "TBD/blah" at sign-off

## Review hierarchy + sign-off

### Preparer
- WP to best standard before submission
- Self-review checklist

### Senior
- Technical accuracy, completeness of procedures + sample coverage,
  cross-ref integrity, coaching notes addressed + closed

### Manager
- Risk + response linkage, judgement appropriateness, section
  conclusions + aggregation, interaction with other sections, materiality
  + misstatement aggregation

### Engagement Partner
- Overall assessment, significant judgements + estimates, GC, fraud +
  management override, auditor's report, closing meeting, written reps

### EQR (ISQM 2)
- Objective evaluation before report signing; judgements + conclusions;
  FS + draft report; selected WPs on significant areas; documented
  sign-off

### Sign-off workflow
- WP: preparer + reviewer stamps
- Lead schedule: manager + EP stamps
- Completion: EP + EQR before report date
- Archive lock = immutability

## Quality control

### Engagement-level (ISA 220 revised)
EP responsibility; team competence; direction + supervision + review
documented; consultation logged; differences of opinion resolved.

### Firm-level (ISQM 1)
Acceptance + continuance; engagement performance monitoring; RCA on
findings; remediation; annual evaluation.

### External assurance
FRC AQR / PCAOB / IAASB inspections; internal monitoring (another
partner); peer review.

## Common FRC AQR / PCAOB deficiencies

- JE testing on sample when full-population available
- GC accepted base case without stressing
- Impairment — no auditor sensitivity, no range
- ECL — model docs only, no back-testing
- Hedge accounting docs prepared after year-end
- Sampling without documented methodology + evaluation
- Walkthroughs refreshed from PY without confirming unchanged
- Estimates — no auditor range, no bias challenge
- Disclosure completeness tick-box without reading the policy
- SUM aggregation inconsistent across group / components
- WPs with incomplete conclusions ("no issues noted")
- Post-archive amendments without documented reason
- EQR sign-off after report date — regulatory breach

## Output pattern

When asked to draft / review / QC any audit-file section:
1. State section + purpose + ISA + assertion
2. Show lead schedule
3. Outline procedures + evidence obtained
4. Surface judgements + estimates
5. State conclusion + cross-references
6. Sign-off layer appropriate to task
7. Flag anti-patterns + quality risks

Cite the Big 4 system used but keep substance system-agnostic.

## Anti-patterns

- WPs not self-contained
- Tickmarks without meaning
- Conclusions that don't address the stated objective
- Review notes closed without preparer action
- Sign-offs before evidence complete
- Reviewing with the report already drafted (anchoring)
- Archive > 60 days without dated reason
- Same-person preparer + reviewer on material WPs
- Stale cross-references after restructure
